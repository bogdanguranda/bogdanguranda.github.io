Trading Assistant

Desktop trading assistant built with Python and PyQt6 for managing trade data, visualizing historical markets, and exporting reporting-ready output. The app is designed to support broker transaction imports, chart-based trade analysis, configurable scanners, and multiple export targets while keeping settings and secrets persisted locally.

What it does

  • Import and normalize transactions from different broker formats
  • Build historical charts with OHLC data, portfolio executions, and trade markers
  • Export transformed trade data to CSV and Google Sheets
  • Manage app settings, themes, and API credentials in a local encrypted store
  • Run background tasks safely using PyQt worker threads for downloads, scans, and exports

What it will be able to do

When fully implemented, the app aims to provide:

  • a complete transaction import/export pipeline for brokers like Alpaca, Interactive Brokers, TradeZero, and others
  • a historical data explorer with layered chart overlays and execution markers
  • an extendable scanner and strategy builder for screening symbols and backtesting ideas
  • seamless Google Sheets integration for sharing reports and trade summaries
  • persistent local settings and encrypted secret storage for API keys

How to run

1. Install Python version specified in .python-version file (you can use pyenv for that)

brew install pyenv

pyenv install 3.13.9
pyenv local 3.13.9 # Sets it for this project

poetry env use 3.13.9
poetry lock && poetry install

2. Create full app

poetry run pyinstaller live.spec --noconfirm

Run from: dist/Trading Assistant/Trading Assistant.exe

3. Upload to Github releases with naming convention, example:

trading-assistant-v3.0.0.zip

Architecture

  • UI Framework: PyQt6 desktop interface with modular tabs and reusable components
  • Plotting: pyqtgraph and Plotly are used for interactive chart rendering
  • Data Processing: pandas and pyarrow for tabular transformation, Excel support via openpyxl
  • Broker APIs: Alpaca (alpaca-py) and Interactive Brokers (ibapi) integration paths
  • Web data: requests and BeautifulSoup for fundamentals and web-based data scraping
  • Persistence: SQLite for settings, plus AES encryption for secrets using cryptography
  • Packaging: PyInstaller to build a self-contained desktop distributable

Key Features

  • Data Transformation: Import, normalize, and export broker transactions
  • Historical Charts: Visualize OHLC data and trade execution markers
  • Theme Support: Multiple themes with dynamic switching
  • Settings Management: Observable models with automatic database persistence
  • Encrypted Secrets: AES-256-CBC encryption for API keys and sensitive data
